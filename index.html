// The Idea

A GPU is not a physical thing. It's a rule.

Thousands of cores doing simple math simultaneously on a grid of data. That's all it is. Strip away the silicon, the transistors, the electrons — and what remains is the math. A function, a grid, and a rule that says: apply simultaneously.

"You don't simulate the things — you simulate the rules that produce the things."

PureBee is proof of that claim. Electricity is just the mechanism that makes the math fast. PureBee replaces the hardware with a specification. The specification replaces silicon with software. The software runs on anything.

If a node can run Node.js ≥ 20, it can run PureBee. No exceptions, no asterisks, no special hardware required.

// Architecture

Four layers. Clean boundaries. No hidden state.

Runtime
llama-streaming.js
Model execution, KV-cache, tokenization, layer streaming. Implements LLaMA architecture using only PureBee instructions. Streams weights from disk — peak memory 1.8GB for a 4.5GB model.
Instruction Set
purebee.js
Named operations: TENSOR_MUL, SOFTMAX, RMS_NORM, ATTENTION. Hardware-agnostic by design — describes what, not how. A transformer can be written using only these operations.
Engine
engine.js · wasm-q4.js
WASM SIMD kernels, Q4 quantized matvec. The binary is constructed byte-by-byte in JavaScript at runtime — no Emscripten, no compiler, no build step. The entire compute stack is readable source.
Memory
memory.js
Tensor layout, named grid registry, quantized storage. Operations address tensors by name, not pointer. gpu.FREE('name') — explicit, auditable, no GC surprises.
// Quickstart
bash
$ git clone https://github.com/PureBee/purebee
$ cd purebee
$ node download.js llama3 # ~770MB
$ node --max-old-space-size=4096 chat-llama3.js

Requires Node.js ≥ 20. The --max-old-space-size flag is not optional — Llama 3.2 1B weights won't fit in Node's default heap.
